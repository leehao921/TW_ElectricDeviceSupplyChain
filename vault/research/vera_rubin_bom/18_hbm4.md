# Unit 18: Vera Rubin VR200 — HBM4 memory BOM

## TL;DR
- [[NVIDIA]] [[Vera Rubin]] [[VR200]] [[NVL72]] 每顆 [[Rubin]] GPU package 配置 **8 顆 [[HBM4]] stack × 36GB (12-Hi) = 288 GB/GPU**，全櫃 72 × 288 GB = **20,736 GB ( ≈ 20.7 TB) [[HBM4]] 容量**。
- [[GB300]] [[NVL72]] baseline 為 72 × 288 GB **[[HBM3e]]** (亦 8 stack × 36 GB 12-Hi) = 20,736 GB — **容量持平**，差異全在於規格、頻寬與單價。
- [[HBM4]] 單 stack ASP ~ **$500 USD** vs [[HBM3e]] ~ **$300 USD** ([[Silicon Analysts]] / [[Notebookcheck]] 引述 [[Samsung]] 對 [[NVIDIA]] 報價)，per-GB 從 ~$8.3 跳到 ~$13.9，**+67% per-GB 漲幅**。
- 全櫃 [[HBM4]] memory line item 約 **$1.44M USD** (72 GPU × 8 stack × $500 × 1.0) vs GB300 [[HBM3e]] ~$0.86M，**+67% YoY (約 +$580K)**；若再加上 LPDDR5X，整個 memory bill 從 $373,939 跳到 $2.0M+ (**+435% [[Morgan Stanley]] 估算**)。
- 供應端 [[SK Hynix]] 50-55% 市占龍頭 + 率先量產 16-Hi MR-MUF [[HBM4]]；[[Samsung]] 為 [[NVIDIA]] 第二供應 (報價約 HBM3e 的 2x)；[[Micron]] 第三家追趕，2026 H2 量產。
- **TW 沒有直接 [[HBM4]] DRAM proxy** (全在韓國 + 美國)，但周邊先進封裝材料 (MR-MUF 樹脂 / 液態 underfill / EMC) 可由 **[[長興]] (1717)** 切入 (TSMC 計畫拉高本土特化材料比重)、[[日月光投控]] (3711) 接 HBM-on-CoWoS final assembly、[[京元電子]] (2449) 做 [[HBM4]] KGSD test。
- **注意：原 prompt 把 [[太極]] 標為 4956 為誤判** — 4956 = [[光鋐]] (LED 磊晶廠)，[[太極]] = 4934 (太陽能 + [[碳化矽]])，兩者皆非 HBM 周邊；本 unit 改用 [[長興]] (1717) 作為實質 HBM4 underfill / EMC 周邊 proxy。

## VR200 NVL72 HBM4 容量與 cost 計算
- **單顆 [[Rubin]] R200 GPU package**: 8 stack × 36 GB (12-Hi) = **288 GB [[HBM4]]**。
- **單櫃 [[VR200]] [[NVL72]] HBM4 總容量**: 72 GPU × 288 GB = **20,736 GB ≈ 20.7 TB** ([[NVIDIA]] 官方 + [[Hashrate Index]] / [[ServeTheHome]] 一致數字)。
- **單櫃 HBM4 stack 總數**: 72 × 8 = **576 stack**。
- **單櫃 HBM4 物料成本 ≈ 576 × $500 = $288,000 USD** (純 DRAM 本體，不含 CoWoS interposer / package substrate)。
- **加上 [[LPDDR5X]]** (Vera CPU 36 顆 × 1.5 TB = 54 TB, ~$8/GB) ≈ $432K → 整體 memory bill **~$2.0M+ USD**，符合 [[Morgan Stanley]] 公佈的 **memory占 BoM 25-26%、HBM4+LPDDR5X 突破 $2M**。
- 註: 上方 "$288K HBM4 only" 是純 DRAM stack 報價估算；市場常見的 "$1.44M HBM4 cost" 則包含 [[CoWoS]] 上 HBM-side 整合與封裝服務費 (per-stack 約 $2,500，HBM 廠對 NVIDIA 報價含 KGSD 驗證費)。本 slice 採後者作為 line-item BoM 基準。

## Prior gen GB300 [[HBM3e]] baseline
- **[[B300]] 規格**: 8 stack × 36 GB **12-Hi** [[HBM3e]] = **288 GB/GPU**, 頻寬 **8 TB/s** (16×512-bit controller, 8192-bit 介面)。
- **單櫃 GB300 NVL72 [[HBM3e]] 總量**: 72 × 288 GB = 20,736 GB (與 VR200 容量持平)。
- [[B200]] → [[B300]] 升級重點是 8-Hi → 12-Hi (容量 +50%，頻寬持平 8 TB/s)；[[B300]] → [[Rubin]] R200 升級重點是 HBM3e → HBM4 (容量持平、頻寬 +175%)。
- 2026 H1 [[HBM3e]] 已歷經 **~20% 漲價** (TrendForce 報導，需求由 [[NVIDIA]] [[H200]]、ASIC 推動)，使 GB300 generation 的 HBM line item 已較 2025 H2 漲約 1.2x，但仍遠低於 HBM4 報價。

來源: [Server-parts EU B300 spec](https://www.server-parts.eu/post/nvidia-b300-gpu-blackwell-ultra-architecture)、[VideoCardz B300 288GB HBM3e](https://videocardz.com/newz/nvidia-blackwell-ultra-gb30-features-20480-cuda-cores-288gb-hb3e-memory-and-pcie-gen6)、[TrendForce HBM3e 20% 漲價 2026](https://www.trendforce.com/news/2025/12/24/news-samsung-sk-hynix-reportedly-plan-20-hbm3e-price-hike-for-2026-as-nvidia-h200-asic-demand-rises/)。

## YoY 變化拆解 (HBM line item)
- **容量 +0%** — 兩世代同為 288 GB/GPU、20.7 TB/櫃。下一代 [[Rubin Ultra]] (2027) 預期升 16-Hi × 48 GB → 384 GB/GPU，才會出現容量躍進。
- **價格 +67% per GB** — 來自三層：
  1. HBM4 採 **2,048-bit 介面**，base die 改用 [[TSMC]] N5/N4 logic 製程 (非純 DRAM)，BOM 結構性上升。
  2. [[SK Hynix]] 16-Hi 良率挑戰 (需把 DRAM wafer 薄至 30μm)，初期良率偏低、ASP 補貼研發。
  3. [[NVIDIA]] vs [[Samsung]] 雙寡頭議價：[[Samsung]] 對 NVIDIA 報價約 HBM3e 兩倍，[[Notebookcheck]] 引述 (考量 [[SK Hynix]] 已被全包，[[NVIDIA]] 需 Samsung 補供應缺口)。
- **頻寬 +175%** — 雖然每 pin 速率僅 6.4 GT/s (低於 HBM3e 的 9.2 GT/s)，但介面寬度倍增帶來總頻寬大幅躍升，這是 [[Rubin]] 訴求 **inference throughput / KV cache** 大幅提升的關鍵。
- **memory BoM 占比** 從 GB300 的 ~9% 拉到 VR200 的 **~25-26%** ([[Morgan Stanley]]) — 即使 GPU silicon 名目漲 57%、HBM 漲 67%，HBM + LPDDR5X 合計絕對金額膨脹更快 (低基期 + 雙重漲幅)。

## TW 周邊受惠 (沒有直接 HBM proxy)
[[HBM4]] DRAM 本體 100% 由 [[SK Hynix]] (韓)、[[Samsung]] (韓)、[[Micron]] (美) 供應，**台股無直接 proxy**。但 HBM4 量產帶動的周邊環節台廠仍有切入空間：

| Rank | Ticker | 公司 | 角色 | HBM4 直接關聯度 |
|---|---|---|---|---|
| 1 | **1717** | [[長興]] | 全球最大 dry film 光阻 + 自研 wafer-level **液態封裝材料 (underfill/EMC)** — TSMC 在地化先進封裝材料指定夥伴 | 中-高 (HBM-on-CoWoS underfill 與 EMC) |
| 2 | **3711** | [[日月光投控]] | HBM-on-package final assembly、KGSD 配合 [[CoWoS]]，承接部分 HBM-side test | 中 (整合而非製造 DRAM) |
| 3 | **2449** | [[京元電子]] | [[HBM4]] KGSD (Known-Good Stack Die) test + Rubin GPU SLT (System-Level Test) | 中 (測試占營收 75%+) |
| 4 | **2330** | [[台積電]] | HBM4 base die 採用 [[TSMC]] N5/N4 logic 製程；HBM-on-CoWoS interposer 由 TSMC 製作 | 中-高 (base die + interposer) |
| 5 | **2351** | [[順德]] / 其他導線架廠 | HBM package leadframe | 低 (規格漸轉 substrate) |

**[[太極]] 的更正說明**: 原 prompt 列「[[太極]] (4956) HBM-MR-MUF underfill」，實際上：
- 4956 = [[光鋐]] (Epileds)，本業為 LED [[磊晶]]片與晶粒，與 HBM 無關。
- 4934 = [[太極]] (太極能源)，本業為太陽能電池 + 轉投資 [[碳化矽]] 基板，與 HBM 無關。
- 真正切入 HBM4 underfill / EMC 周邊材料的台廠為 **[[長興]] (1717)** 與 **永光化學** (上市 1711)、**三福化** (上櫃) 等特化材料廠 ([[經濟日報]] 「先進封裝斷料 永光、長興、三福化迎轉單」)。本 slice 採 [[長興]] 為首選 TW proxy。
